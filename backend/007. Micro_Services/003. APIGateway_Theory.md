# API Gateway

- Entry point for all client requests. Handles routing, authentication, rate limiting, and security before forwarding to
  internal services.

## Key Responsibilities

- Authentication & Authorization (JWT, OAuth)
- Rate limiting (prevent abuse / DDoS)
- Routing (path-based / service-based)
- Request/Response transformation
- Logging & audit (critical in banking)

## Mostly used

- Modern/cloud-native → Kong, AWS API Gateway
- Internal microservices → Spring Cloud Gateway

## Blunt Gaps Most People Miss

- Gateway is a bottleneck + single point of failure
- Latency increases (extra hop)
- Misconfigured rate limits → production outages
