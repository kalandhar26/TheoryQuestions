## The Pipeline Variable: `$_` and `$PSItem`

**Understanding the Pipeline Variable**

- The `$_` (or $PSItem in PowerShell 3+) represents the current object in the pipeline.

```powershell
# Both are identical
Get-Process | ForEach-Object { $_.Name }
Get-Process | ForEach-Object { $PSItem.Name }

# Common aliases
Get-Process | % { $_.Name }           # % is alias for ForEach-Object
Get-Process | foreach { $_.Name }     # foreach is also an alias

# Advanced Pipeline Variable Usage

# Accessing nested properties with null-conditional operator (PS7+)
Get-Service | ForEach-Object {
    $container = $_.Site?.Container ?? 'N/A'
    [PSCustomObject]@{
        Name = $_.Name
        SiteContainer = $container
    }
}

# Multiple property access in single pipeline
Get-Process | Where-Object {
    $_.CPU -gt 100 -and $_.WorkingSet64 -gt 500MB
} | ForEach-Object {
    "$( $_.ProcessName ) using $( $_.WorkingSet64/1MB ) MB CPU: $( $_.CPU )"
}
```

## ForEach-Object: Three Syntax Forms

**Script Block Syntax (Full Control)**

```powershell
Get-Process | ForEach-Object {
    # Begin block (runs once before processing)
    $totalCPU = 0
} {
    # Process block (runs for each object)
    $totalCPU += $_.CPU
    [PSCustomObject]@{
        Name = $_.ProcessName
        CPUPercent = $_.CPU
    }
} {
    # End block (runs once after all objects)
    Write-Host "Total CPU used: $totalCPU"
}
```

**Simplified Syntax (Property/Method Access)**

```powershell
# Get property value directly (PS3+)
Get-Process | ForEach-Object ProcessName

# Call method directly
Get-Process | ForEach-Object Kill          # Dangerous but valid!

# Equivalent to:
Get-Process | ForEach-Object { $_.Kill() }
```

**Parallel Processing**

```powershell
# Process items in parallel (use for long-running operations only!)
$computers = 'Server01', 'Server02', 'Server03', 'Server04'
$computers | ForEach-Object -Parallel {
    Test-Connection -TargetName $_ -Count 2 -Quiet
} -ThrottleLimit 4

# Using external variables with $using:
$timeout = 30
$servers | ForEach-Object -Parallel {
    Test-Connection -TargetName $_ -TimeoutSeconds $using:timeout
} -ThrottleLimit 10 -TimeoutSeconds 60

# ⚠️ Important: -PipelineVariable does NOT work in parallel mode
```

- Parallel has overhead. Only use when operation takes >1 second or is I/O bound.

### Filtering with Where-Object

```powershell
# Script block syntax
Get-Process | Where-Object { $_.CPU -gt 100 }

# Simplified syntax (PS3+)
Get-Process | Where-Object CPU -gt 100

# Multiple conditions
Get-Process | Where-Object { $_.CPU -gt 100 -and $_.WorkingSet64 -gt 500MB }

# Filter with calculated properties
Get-ChildItem | Where-Object {
    ($_.LastWriteTime -gt (Get-Date).AddDays(-7)) -and
            ($_.Extension -match '\.(log|txt)$')
}

# Using Where method on collections (faster for in-memory arrays)
$processes = Get-Process
$processes.Where({ $_.CPU -gt 100 }, 'Default')   # All matching
$processes.Where({ $_.CPU -gt 100 }, 'First')     # First match only
$processes.Where({ $_.CPU -gt 100 }, 'Last')      # Last match only
$processes.Where({ $_.CPU -gt 100 }, 'SkipUntil') # Skip until match
$processes.Where({ $_.CPU -gt 100 }, 'Until')     # Return until match
```

## Advanced Parameter Binding: ByValue vs ByPropertyName

**How Pipeline Binding Works?**

- PowerShell attempts parameter binding in this order:

        ByValue (type matching)
        ByPropertyName (name matching)

**ByValue Binding**

- The entire object is passed to a parameter that accepts the same type

```powershell
# Stop-Process accepts Process objects ByValue via -InputObject
Get-Process notepad | Stop-Process

# Equivalent to:
Stop-Process -InputObject (Get-Process notepad)
```

**ByPropertyName Binding**

- Object properties bind to parameters with matching names

```powershell
# CSV with Name column binds to Stop-Process -Name
Import-Csv processes.csv | Stop-Process
# CSV: Name
#      notepad
#      calc

# Custom object with matching property names
$serviceData = @(
    [PSCustomObject]@{ Name = 'w32time' }
    [PSCustomObject]@{ Name = 'bits' }
)
$serviceData | Stop-Service  # Binds Name property to -Name parameter
```

**Creating Pipeline-Compatible Functions**

```powershell
function Get-FileSize
{
    [CmdletBinding()]
    param(
        [Parameter(
                Mandatory = $true,
                ValueFromPipelineByPropertyName = $true, # Accepts Path property
                ValueFromPipeline = $true                  # Also accepts FileInfo objects
        )]
        [Alias('FullName')]  # Alternative property name mapping
        [string]$Path
    )

    process {
        foreach ($p in $Path)
        {
            $item = Get-Item $p
            [PSCustomObject]@{
                Name = $item.Name
                Path = $item.FullName
                SizeMB = [math]::Round($item.Length / 1MB, 2)
                Modified = $item.LastWriteTime
            }
        }
    }
}

# Usage scenarios:
Get-ChildItem *.txt | Get-FileSize           # ByValue (FileInfo has FullName)
Import-Csv files.csv | Get-FileSize           # ByPropertyName (CSV has Path column)
Get-FileSize -Path "C:\data\file.txt"        # Direct parameter
```

## -PipelineVariable

- The `-PipelineVariable` (alias -PV) stores the current pipeline object for reuse in later pipeline stages

```powershell
# Without -PipelineVariable (re-evaluates properties)
Get-Process | Where-Object { $_.CPU -gt 100 } |
        Select-Object Name, @{ N = 'CPU'; E = { $_.CPU } }

# With -PipelineVariable (stores object, avoids re-evaluation)
Get-Process -PipelineVariable proc |
        Where-Object { $proc.CPU -gt 100 } |
        Select-Object Name, @{ N = 'CPU'; E = { $proc.CPU } }

# Complex example: Access original object after transformation
Get-Service -PipelineVariable svc |
        Where-Object { $svc.Status -eq 'Running' } |
        ForEach-Object {
            [PSCustomObject]@{
                ServiceName = $svc.Name           # Original service object
                DisplayName = $svc.DisplayName
                Status = $_.Status                # Current object (same here)
                StartType = $svc.StartType
            }
        }
```

### Begin/Process/End Blocks

- Advanced functions must use these blocks for proper pipeline handling.

```powershell
function Measure-Sum
{
    [CmdletBinding()]
    param(
        [Parameter(ValueFromPipeline = $true)]
        [int]$Number
    )

    begin {
        # Runs ONCE before pipeline starts
        Write-Host "Starting calculation..." -ForegroundColor Green
        $script:total = 0
        $script:count = 0
    }

    process {
        # Runs for EACH object in pipeline
        $script:total += $Number
        $script:count++
        Write-Verbose "Processed $Number, running total: $script:total"
    }

    end {
        # Runs ONCE after pipeline completes
        [PSCustomObject]@{
            Count = $script:count
            Sum = $script:total
            Average = $script:total / $script:count
        }
        Write-Host "Calculation complete!" -ForegroundColor Green
    }
}

# Usage
1..10 | Measure-Sum -Verbose
```

| Block     | Execution Count | Use Case                               |
|-----------|-----------------|----------------------------------------|
| `begin`   | Once            | Initialize variables, open connections |
| `process` | Per object      | Process each pipeline item             |
| `end`     | Once            | Cleanup, return final results          |

## Troubleshooting Parameter Binding

**Trace-Command (Debugging Pipeline Binding)**

```powershell
# See exactly how PowerShell binds parameters
Trace-Command -Name ParameterBinding -Expression {
    Get-Process | Stop-Process
} -PSHost

# Check if parameter accepts pipeline input
Get-Help Stop-Process -Parameter * |
        Where-Object { $_.pipelineInput -match 'True' } |
        Select-Object Name, pipelineInput
```

**Common Binding Issues and Solutions**

```powershell
# Problem: Property name mismatch
$data = [PSCustomObject]@{ FilePath = 'C:\test.txt' }
# $data | Get-Content  # FAILS - Get-Content expects -Path, not -FilePath

# Solution 1: ForEach-Object explicit mapping
$data | ForEach-Object { Get-Content -Path $_.FilePath }

# Solution 2: Select-Object property renaming
$data | Select-Object @{ N = 'Path'; E = { $_.FilePath } } | Get-Content

# Solution 3: Alias in function parameter (shown in section 4)
```

**Pipeline Performance Optimization**

| Approach            | Memory           | Speed   | Best For                  |
|---------------------|------------------|---------|---------------------------|
| `foreach` statement | High (loads all) | Fastest | In-memory collections     |
| `ForEach-Object`    | Low (streaming)  | Slower  | Large datasets, pipelines |
| `.ForEach()` method | High (in-memory) | Fast    | Method chaining on arrays |

```powershell
# Fastest for in-memory (but loads all into RAM)
$processes = Get-Process
foreach ($proc in $processes)
{
    $proc.Name
}

# Most memory efficient (streaming)
Get-Process | ForEach-Object { $_.Name }

# Method syntax (PowerShell 5+)
(Get-Process).ForEach({ $_.Name })

# Parallel (for long operations only!)
Get-Process | ForEach-Object -Parallel { $_.Name } -ThrottleLimit 5
```

## Real World Example

```powershell
<#
.SYNOPSIS
    Advanced pipeline demonstration with all techniques combined.
#>
function Get-SystemReport
{
    [CmdletBinding()]
    param(
        [Parameter(ValueFromPipeline = $true, ValueFromPipelineByPropertyName = $true)]
        [Alias('ComputerName', 'Server')]
        [string[]]$Name,

        [int]$CpuThreshold = 80,

        [switch]$IncludeServices
    )

    begin {
        $reportData = [System.Collections.Generic.List[hashtable]]::new()
        Write-Host "Generating system reports..." -ForegroundColor Cyan
    }

    process {
        foreach ($computer in $Name)
        {
            try
            {
                # Test connection with pipeline variable to reuse connection test
                Test-Connection -TargetName $computer -Count 1 -ErrorAction Stop -PipelineVariable conn |
                        ForEach-Object {
                            # Create CIM session for reuse
                            $cimSession = New-CimSession -ComputerName $computer

                            # Get system info using pipeline variable for computer name
                            $os = Get-CimInstance -CimSession $cimSession -ClassName Win32_OperatingSystem
                            $cpu = Get-CimInstance -CimSession $cimSession -ClassName Win32_Processor
                            $processes = Get-Process -ComputerName $computer -ErrorAction SilentlyContinue |
                                    Where-Object { $_.CPU -gt $CpuThreshold }

                            $systemInfo = [PSCustomObject]@{
                                ComputerName = $computer
                                Status = 'Online'
                                OS = $os.Caption
                                TotalMemoryGB = [math]::Round($os.TotalVisibleMemorySize / 1MB, 2)
                                CPUUsage = $cpu.LoadPercentage
                                HighCPUProcesses = $processes | Select-Object -First 5 Name, CPU
                                Services = if ($IncludeServices)
                                {
                                    Get-Service -ComputerName $computer |
                                            Where-Object { $_.Status -eq 'Running' } |
                                            Select-Object -First 10 Name, DisplayName
                                }
                                else
                                {
                                    $null
                                }
                                Timestamp = Get-Date
                            }

                            $reportData.Add($systemInfo)
                            Remove-CimSession $cimSession
                        }
            }
            catch
            {
                $reportData.Add([PSCustomObject]@{
                    ComputerName = $computer
                    Status = 'Offline'
                    Error = $_.Exception.Message
                    Timestamp = Get-Date
                })
            }
        }
    }

    end {
        # Group by status for summary
        $summary = $reportData | Group-Object -Property Status |
                Select-Object Name, Count, @{ N = 'Systems'; E = { $_.Group.ComputerName -join ', ' } }

        [PSCustomObject]@{
            Summary = $summary
            Details = $reportData
            GeneratedAt = Get-Date
            TotalSystems = $reportData.Count
        }
    }
}

# Usage demonstrating all pipeline techniques
$servers = @(
[PSCustomObject]@{ ComputerName = 'Server01' }
[PSCustomObject]@{ ComputerName = 'Server02' }
[PSComputerName = 'Server03'
}  # Different property name
)

# ByPropertyName binding handles all three object types!
$servers | Get-SystemReport -CpuThreshold 50 -IncludeServices |
Select-Object -ExpandProperty Details |
Where-Object {
$_.Status -eq 'Online'
} |
ForEach-Object -Parallel {
# Export each to CSV in parallel
$_ | Export-Csv "C:\Reports\$($_.ComputerName).csv" -NoTypeInformation
} -ThrottleLimit 3
```
