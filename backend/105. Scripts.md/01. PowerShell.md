# PowerShell

## Introduction

- **Question:** You have just wrapped up a long, exhausting day only to find that your computer is running agonizingly
  slowly.
- **Traditional Ways:** Open the Task Manager, toggle between tabs and play detective until you pinpoint the issue.
  That's not a terrible plan, but let's be honest, we can do better.
- **Modern Ways:** Here is where PowerShell comes to the rescue. With just a few keystrokes we can access vital system
  information and figure out what's slowing down our computer.
- One of the common culprits for a slower system is resource-heavy processes.

```sh
Get-Process | Sort-Object CPU -Descending | Select-Object -First 5
```

- The above command simply lists the top five processes consuming the most CPU resources at this moment.

## Using PowerShell for System Performance Diagnostics

- Diagnosing performance issues is often the first step for any IT professional core for maintaining healthy systems.
  While tools like Task Manager make this easy on a single machine, they quickly become impractical when you are
  responsible for hundreds or even thousands of systems. This is where PowerShell becomes invaluable.
- PowerShell enables you to automate performance diagnostics and run commands remotely across multiple systems. Instead
  of manually checking each machine, you can gather performance data at scale with just a few commands.
- Whether you are a power user looking to boost productivity or an IT professional managing a large enterprise
  environment, PowerShell provides a level of flexibility, efficiency, and control that traditional graphical interfaces
  cannot match.

---

## Understanding CPU Time

- CPU Time represents the total amount of processor time a process has consumed since it started running. If a process
  shows 1 minute of CPU time, it means that across its entire runtime, it has used the CPU for a cumulative total of one
  minute.
- This metric is especially useful for identifying processes that are consuming excessive CPU resources over time.

## PowerShell Pipelines Explained

- In PowerShell, the pipe symbol (`|`) is used to create what is known as a pipeline.
- The pipeline allows the output of one cmdlet to be passed directly as input to another cmdlet. This enables powerful
  command chaining and data processing.
- For example: `Get-Process` might be the first step in a pipeline. Its output can then be filtered, sorted, or
  formatted by subsequent cmdlets.
- Pipelines are one of PowerShell's most powerful features, allowing you to build efficient, readable, and highly
  reusable command sequences.

## PowerShell Cmdlets

- Cmdlets are simple, small, focused programs designed to perform a specific task.
- They are the heart and soul of PowerShell and understanding them is crucial to mastering this powerful tool.

### PowerShell Cmdlet Naming Convention

- PowerShell cmdlets follow a standard naming convention to ensure consistency and readability across the platform.

**Verb-Noun Format**

Each cmdlet name is composed of:

- A **verb** (PowerShell uses a predefined set of approved verbs to describe the action a cmdlet performs. This helps
  maintain consistency across modules and scripts.)
- A **noun** (The noun portion of the cmdlet name identifies the object or resource the cmdlet acts upon.)
- Separated by a hyphen (`-`)

## PowerShell Parameters

Imagine you are an IT administrator responsible for managing a server. One of your routine tasks is to identify and
analyze specific processes in order to troubleshoot performance or stability issues. PowerShell parameters make this
task precise, efficient, and automation-friendly.

| Cmdlet                                      | Description                                                                                                                                |
|---------------------------------------------|--------------------------------------------------------------------------------------------------------------------------------------------|
| `Get-Process -Name chrome`                  | Retrieves all running processes with the name chrome.                                                                                      |
| `Get-Process -Name chrome -FileVersionInfo` | Displays detailed file version information for each Chrome process, which can be useful for troubleshooting compatibility or update issues |
| `Get-Process -Id 23799`                     | Retrieves a specific process using its Process ID (PID).                                                                                   |

- Finding processes by Process ID (PID) offers several important advantages.

---

### Greater Precision

- Process IDs are unique to each running instance, making this method more accurate than searching by Name — especially
  when multiple instances of the same application are running.

### Guaranteed Accuracy

- Since no two running processes share the same PID, you can be confident that you are targeting the exact process you
  intend to analyze or manage.

### More Reliable for Automation

- In scripts and automated tasks, PIDs are more dependable because they are not affected by:
    - Language or localization settings.
    - Variations in application names.
    - Multiple instances of the same program.

## Optional and Positional Parameters

- PowerShell cmdlets support both optional parameters and positional parameters, giving you flexibility in how you write
  commands.
- Optional parameters are parameters that are not required to run a cmdlet. If you don't specify them, PowerShell uses
  default values or predefined behavior.
- Positional parameters allow you to supply parameter values without explicitly typing the parameter name, as long as
  the values are provided in the correct order.

## PowerShell Parameter Values

- In PowerShell, each parameter expects a specific type of value. Providing the correct data type is essential for
  commands to execute successfully.

### Common Parameter Value Types

- **Numeric Values** — Some parameters require numbers.
- **String Values** — Other parameters expect string values, which should be enclosed in quotes when they contain text.
- **Switch Parameters (No Value Required)** — Not all parameters require a value. Some parameters are known as switch
  parameters. Their presence alone changes the behavior of the cmdlet.

| Cmdlet                                              | Description                                                                                                                                                         |
|-----------------------------------------------------|---------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| `Test-Connection -ComputerName google.com`          | Sends 4 ICMP echo requests (pings) to google.com using the default settings and displays detailed results such as `response time`, `packet loss` and `status`       |
| `Test-Connection -ComputerName google.com -Quiet`   | Tests whether google.com is reachable and returns a Boolean value (True or False) instead of detailed output. This is commonly used in scripts and automated tasks. |
| `Test-Connection -ComputerName google.com -Count 1` | Sends only 1 ICMP echo request instead of the default 4, allowing for a quicker connectivity check with reduced network traffic                                     |

- The default count of 4 is modified to 1, reducing the number of ping requests sent (last command).

## Paths and Locations in PowerShell

### Absolute Paths vs Relative Paths

- PowerShell uses a location-based system, similar to navigating folders in File Explorer. Understanding how paths work
  is essential for efficiently managing files and directories.
- PowerShell allows you to temporarily save and return to locations using a location stack (Push and Pop Locations).

| View Directory              | Cmdlet                            | Description                                                                                                      |
|-----------------------------|-----------------------------------|------------------------------------------------------------------------------------------------------------------|
|                             | `Get-ChildItem`                   | Displays the contents of a directory.                                                                            |
|                             | `Get-ChildItem "C:\Users\"`       | Lists files and folders in the current directory.                                                                |
| **Changing Locations**      |                                   |                                                                                                                  |
| Absolute Paths              | `Set-Location "C:\Program Files"` | Used to move between directories. Moves directly to the specified location, regardless of your current directory |
| Relative Paths              | `Set-Location Java`               | Moves into the Java folder if it exists in the current directory.                                                |
| **Special Path Symbols**    |                                   |                                                                                                                  |
| Current Directory (`.`)     | `Set-Location .\Documents`        | `.` represents the current directory. Moves into the Documents folder located within the current directory.      |
| Parent Directory (`..`)     | `Set-Location ..\`                | Moves one directory level up.                                                                                    |
|                             | `Set-Location ..\..\..\`          | Moves three directory levels up.                                                                                 |
| Home Directory (`~`)        | `Set-Location ~`                  | Navigates to the home directory of the current user.                                                             |
| Push Current Location       | `Push-Location`                   | Saves the current directory onto the stack.                                                                      |
| Return to Previous Location | `Pop-Location`                    | Returns to the last saved location.                                                                              |

### Symbolic Links (SymLinks)

**Why Use Symbolic Links?**

- You need the same file or folder accessible from multiple locations.
- Copying files wastes disk space.
- Moving files breaks scripts or applications.

**Solution:**

- A symbolic link solves this problem.
- A symlink is a special file that points to another file or folder.
- When you access the symlink, the system automatically redirects you to the target.

**Creating a SymLink:**

```sh
New-Item -ItemType SymbolicLink -Path "Documents\MySymLink" -Target "Desktop\Original_Folder"
```

- Works with files and directories.
- Can span different drives.
- Saves disk space.

**Drawbacks:**

- If the original file or folder is moved or deleted, the symlink becomes broken.

---

### Hard Links

- SymLinks break when the target is moved or deleted. Hard links are more robust.
- A hard link behaves like the original file, points to the same data on disk, and appears as if the file exists in
  multiple locations (Multiple access points).
- Does not consume extra disk space. No broken references if one copy is deleted.
- Remains valid even if the original file is deleted.

**Creating a Hard Link:**

```sh
New-Item -ItemType HardLink -Path "Documents\MyHardLink.txt" -Target "Desktop\Original_File.txt"
```

**Use Cases:**

- Multiple project folders require the same configuration file.
- You want to avoid duplicating data.

**Drawbacks:**

- Can only be created for files, not directories.
- Must exist on the same physical drive.

| Use Case                             | Recommended Option   |
|--------------------------------------|----------------------|
| Automating file/folder creation      | `New-Item`           |
| Linking folders or cross-drive paths | Symbolic Link        |
| Maximum reliability for files        | Hard Link            |
| Avoiding duplicate file data         | Symbolic / Hard Link |

### Reading File Contents with Get-Content

- `Get-Content` is used to read the contents of a file. This is useful in reviewing logs or comparing outputs across
  systems.

| Cmdlet                                                           | Description                                                  |
|------------------------------------------------------------------|--------------------------------------------------------------|
| `Get-Content file.txt`                                           | Read file using relative path.                               |
| `Get-Content "C:\Users\file2.txt"`                               | Read file using absolute path.                               |

### Writing to Files with Set-Content / Add-Content

- This enables automated file updates, log file creation, configuration and script generation.
- **`Set-Content`** replaces all existing content in a file. Best for creating new files and completely overwriting
  existing files.
- **`Add-Content`** adds content to the end of a file without removing existing data. Best for logs, journals, and
  incremental updates.

| Cmdlet                                                                                                          | Description                                         |
|-----------------------------------------------------------------------------------------------------------------|-----------------------------------------------------|
| `Set-Content -Path file1.txt -Value "Hello, PowerShell World!"`                                                 | Write content to file by removing existing content. |
| `Add-Content -Path file1.txt -Value "PowerShell rocks!"`                                                        | Write new content at the end of the file.           |
| `Set-Content -Path "C:\Users\file1.txt" -Value "Journal Entry: $(Get-Date -Format 'yyyy-MM-dd')"`               | Create a file and add current date.                 |
| `Add-Content -Path "C:\Users\file1.txt" -Value "Learning 1: Reading file content with Get-Content"`             | Add or append content.                              |
| `Add-Content -Path "C:\Users\file1.txt" -Value "Learning 2: Creating and overwriting content with Set-Content"` | Add or append content.                              |
| `Add-Content -Path "C:\Users\file1.txt" -Value "Learning 3: Appending content with Add-Content"`                | Add or append content.                              |

- This approach is commonly used in system logs, audit trails, and automation scripts.

## Copying and Moving Files in PowerShell

**Why Not Just Drag and Drop?** Yes, you can copy or move files using File Explorer, but:

- You are a system administrator managing hundreds of computers.
- You are a data analyst working with thousands of files.
- You need to repeat the same task every day or across systems.

- In such cases, dragging and dropping files manually becomes slow, error-prone, and inefficient.

- PowerShell allows you to copy, move, and rename files programmatically, making these tasks: Automated, Scalable,
  Consistent, and Script-friendly.

- You can rename files or folders during copy or move operations simply by changing the destination name.

### Core Cmdlets for Copying and Moving

| Cmdlet                                                                                                           | Description                                    |
|------------------------------------------------------------------------------------------------------------------|------------------------------------------------|
| `Move-Item -Path <Source> -Destination <DestinationPath>`                                                        | Move files / folders.                          |
| `Copy-Item -Path <Source> -Destination <DestinationPath>`                                                        | Copy files / folders.                          |
| `Copy-Item -Path "C:\Projects\App\Config.json" -Destination "C:\Backup\Config.json"`                             | Creates a copy of the source file.             |
| `Copy-Item -Path "C:\Users\Old_Folder\" -Destination "C:\Users\Old_Folder_Renamed\"`                             | Rename while copying.                          |
| `Move-Item -Path "C:\Reports\Sales.xlsx" -Destination "C:\Archive\Sales_2026.xlsx"`                              | Rename while moving.                           |
| `$timestamp = Get-Date -Format "yyyyMMdd_HHmmss"; Copy-Item "C:\Data\log.txt" "C:\Archive\log_${timestamp}.txt"` | Migrating user data and appending a timestamp. |
| `Copy-Item data.csv data_ModelA_Trial1.csv`                                                                      | Copying and renaming.                          |
| `Copy-Item -Path "C:\Source" -Destination "C:\Target" -Recurse`                                                  | Copy entire directory tree.                    |
| `Copy-Item -Path file.txt -Destination backup.txt -Force`                                                        | Overwrite existing file.                       |
| `Move-Item -Path "C:\Temp" -Destination "C:\Archive" -WhatIf`                                                    | Test before running.                           |

**Why Copy Files? — Common Use Cases:**

- Creating backups before making major changes.
- Experimenting with new features without touching the original files.
- Distributing datasets for analysis.

**Why Move Files? — Common Use Cases:**

- Migrating user data.
- Reorganizing directories.
- Archiving old files.

### Renaming Items in PowerShell

### Deleting Items in PowerShell

- Deleting files and folders can also be automated using `Remove-Item`.
- Cleaning up temporary files across multiple servers.
- Deleting old log files that consume disk space.
- Removing obsolete user data.
- Automating maintenance tasks.
- Sometimes files cannot be deleted because they are read-only, they are hidden, or permissions are restrictive. We can
  use `-Force`.

| Cmdlet                                                            | Description                                                                 |
|-------------------------------------------------------------------|-----------------------------------------------------------------------------|
| `Remove-Item -Path <ItemToDelete>`                                | Basic delete syntax.                                                        |
| `Remove-Item -Path "C:\Users\Important_Documents"`                | Removes the item only if it is empty.                                       |

| `Remove-Item -Path "C:\Users\Stubborn_Documents" -Force`          | Forcefully delete.                                                          |
| `Remove-Item -Path "C:\Users\Stubborn_Documents" -Recurse -Force` | Forcefully delete entire folder along with its contents.                    |
| `Remove-Item -Path "C:\Temp\*" -Recurse -WhatIf`                  | Shows what would be deleted, prevents acciental data loss                   |

#### Common Deletion Failures (Important)

**PowerShell cannot delete a file if:**

- The file is currently in use or locked by another process.
- You do not have permission to access the file.
- The file is protected by the system.

**In such cases, you may need:**

- Administrative privileges.
- To stop the locking process.
- To take ownership or adjust permissions.

## Archiving and Compressing Items in PowerShell

### Compressing and Decompressing Files and Directories

- PowerShell makes it easy to compress (archive) and decompress (extract) files and folders using built-in cmdlets. This
  is especially useful for automation, backups, and data transfers.
- File compression provides several practical benefits: Saves disk space, Speeds up file transfers (e.g., transferring a
  100 MB ZIP instead of a 1 GB folder), Simplifies file management by bundling related files together.
- File compression is a method of encoding information using fewer bits than the original. Files with repetitive text or
  data usually compress very well.

| Cmdlet                                                                                                                       | Description                               |
|------------------------------------------------------------------------------------------------------------------------------|-------------------------------------------|
| `Compress-Archive -Path <SourcePath> -DestinationPath <ArchivePath>`                                                         | Archive/zip syntax.                       |
| `Compress-Archive -Path "C:\Users\Big_File.txt" -DestinationPath "C:\Users\Smaller_File.zip"`                                | Creates a ZIP archive.                    |
| `Compress-Archive -Path "C:\Users\Big_File1.txt","C:\Users\Big_File2.txt" -DestinationPath "C:\Users\Consolidated_File.zip"` | Add multiple files to one archive.        |
| `Compress-Archive -Path "C:\Users\Big_File3.txt" -DestinationPath "C:\Users\Consolidated_File.zip" -Update`                  | Adds the new file to an existing archive. |

- What if the ZIP file already exists, and you want to add more files? That's where `-Update` comes in.

```sh
1..100000 | ForEach-Object {
  "This is Line $_ of a large, compressible file. It contains repetitive text."
} | Out-File -FilePath "Desktop\Big_File.txt"
```

- The above command will create a large file in the specified directory.

**About ZIP Extensions:**

- PowerShell always creates ZIP archives.
- If you omit the extension, `.zip` is added automatically.
- Even if you use other extensions (`.raw`, etc.), PowerShell still creates a ZIP file internally.

## Extracting Archives with Expand-Archive

| Cmdlet                                                                                              | Description                      |
|-----------------------------------------------------------------------------------------------------|----------------------------------|
| `Expand-Archive -Path "C:\Users\Consolidated_File.zip"`                                             | Extract to current location.     |
| `Expand-Archive -Path "C:\Users\Consolidated_File.zip" -DestinationPath "C:\Users\Unzipped_Folder"` | Extract to specific destination. |
