## Creating Custom Properties with Select-Object

**Calculated Properties (Hashtable Syntax)**

- The most common way to create custom properties in pipelines

```powershell
# Basic syntax: @{Name='PropertyName'; Expression={ScriptBlock}}
Get-Process | Select-Object Name, @{
    Name = 'MemoryMB'
    Expression = { [math]::Round($_.WorkingSet64 / 1MB, 2) }
}

# Shorthand syntax (Label/Expression or l/e)
Get-ChildItem | Select-Object Name, @{
    l = 'SizeKB'
    e = { [math]::Round($_.Length / 1KB, 2) }
}, @{
    l = 'DaysOld'
    e = { ((Get-Date) - $_.LastWriteTime).Days }
}
```

**Advanced Calculated Properties**

```powershell
# Conditional logic in expressions
Get-Service | Select-Object DisplayName, Status, @{
    Name = 'Comments'
    Expression = {
        if ($_.Status -eq 'Running')
        {
            'All good!'
        }
        else
        {
            'Needs attention'
        }
    }
}

# Nested property access with null-checking (PS7+)
Get-Process | Select-Object Name, @{
    Name = 'Company'
    Expression = { $_.Company?.Name ?? 'Unknown' }
}

# Formatting with -f operator
Get-CimInstance Win32_LogicalDisk | Select-Object DeviceID, @{
    Name = 'FreeSpaceGB'
    Expression = { "{0:N2}" -f ($_.FreeSpace / 1GB) }
}, @{
    Name = 'TotalSizeGB'
    Expression = { "{0:N2}" -f ($_.Size / 1GB) }
}
```

## Script Blocks: Begin, Process, End

**The Three Blocks Explained**

- Every pipeline function should use these blocks for proper streaming

```powershell
function Measure-Data
{
    [CmdletBinding()]
    param(
        [Parameter(ValueFromPipeline = $true)]
        [int]$Number
    )

    begin {
        # Runs ONCE before pipeline starts
        # Initialize variables, open connections, prepare resources
        $script:sum = 0
        $script:count = 0
        $script:min = [int]::MaxValue
        $script:max = [int]::MinValue

        Write-Verbose "Starting calculation..."
    }

    process {
        # Runs for EACH object in pipeline
        # Process individual items here
        $script:sum += $Number
        $script:count++
        if ($Number -lt $script:min)
        {
            $script:min = $Number
        }
        if ($Number -gt $script:max)
        {
            $script:max = $Number
        }

        # Can also output transformed objects
        [PSCustomObject]@{
            Input = $Number
            RunningSum = $script:sum
        }
    }

    end {
        # Runs ONCE after all objects processed
        # Cleanup, return final results, close connections
        [PSCustomObject]@{
            Count = $script:count
            Sum = $script:sum
            Average = $script:sum / $script:count
            Minimum = $script:min
            Maximum = $script:max
        }

        Write-Verbose "Calculation complete!"
    }
}

# Usage
1..100 | Measure-Data -Verbose
```

**Process Block Variations**

```powershell
# Multiple parameters with pipeline input
function Set-Configuration
{
    [CmdletBinding()]
    param(
        [Parameter(ValueFromPipeline = $true, ValueFromPipelineByPropertyName = $true)]
        [string]$ComputerName,

        [Parameter(ValueFromPipelineByPropertyName = $true)]
        [int]$Port = 443,

        [Parameter()]
        [string]$Environment = 'Production'
    )

    process {
        # Process block executes for each pipeline object
        foreach ($computer in $ComputerName)
        {
            [PSCustomObject]@{
                Computer = $computer
                Port = $Port
                Environment = $Environment
                Configured = $true
            }
        }
    }
}

# Handles both pipeline and direct input
$data = @(
    [PSCustomObject]@{ ComputerName = 'Server01'; Port = 8080 }
    [PSCustomObject]@{ ComputerName = 'Server02' }  # Uses default Port
)
$data | Set-Configuration -Environment 'Development'
```

### Add-Member: Dynamic Property Addition

**Basic Add-Member Usage**

```powershell
# Create base object
$service = Get-Service w32time

# Add NoteProperty (static value)
$service | Add-Member -MemberType NoteProperty -Name 'CustomID' -Value 'SRV-001' -PassThru

# Add ScriptProperty (dynamic calculation)
$service | Add-Member -MemberType ScriptProperty -Name 'Age' -Value {
    (Get-Date) - $this.StartTime
} -PassThru

# Add AliasProperty (shortcut)
$service | Add-Member -MemberType AliasProperty -Name 'SvcName' -Value 'ServiceName' -PassThru

# Add ScriptMethod (action)
$service | Add-Member -MemberType ScriptMethod -Name 'Restart' -Value {
    $this.Stop()
    Start-Sleep 2
    $this.Start()
} -Force
```

**Chaining Add-Member (PassThru)**

```powershell
# Create object and chain properties
$server = [PSCustomObject]@{ Name = 'Server01' } |
        Add-Member -MemberType NoteProperty -Name 'IP' -Value '192.168.1.1' -PassThru |
        Add-Member -MemberType NoteProperty -Name 'Role' -Value 'Web' -PassThru |
        Add-Member -MemberType ScriptProperty -Name 'Status' -Value {
            if (Test-Connection -TargetName $this.Name -Count 1 -Quiet)
            {
                'Online'
            }
            else
            {
                'Offline'
            }
        } -PassThru

# Now $server has all properties
$server | Format-List *
```

**Performance Warning: Add-Member vs PSCustomObject**

- Add-Member is significantly slower than creating objects with [PSCustomObject]

| Method                              | Performance | Use Case                   |
|-------------------------------------|-------------|----------------------------|
| `[PSCustomObject]@{}`               | **Fastest** | Creating new objects       |
| `Select-Object`                     | Fast        | Shaping existing objects   |
| `New-Object PSObject -Property @{}` | Moderate    | Legacy compatibility       |
| **Add-Member**                      | **Slowest** | Adding to existing objects |

```powershell
# BAD: Slow for many objects
$results = @()
foreach ($item in $data)
{
    $obj = New-Object PSObject
    $obj | Add-Member Name $item.Name
    $obj | Add-Member Value $item.Value
    $results += $obj  # Also inefficient!
}

# GOOD: Fast object creation
$results = foreach ($item in $data)
{
    [PSCustomObject]@{
        Name = $item.Name
        Value = $item.Value
        Calculated = $item.Value * 2
    }
}
```

## Update-TypeData: Persistent Type Extensions

**Why Use Update-TypeData?**

- Unlike Add-Member (instance-only), Update-TypeData modifies the type definition—affecting all objects of that type.

```powershell
# Add properties to ALL FileInfo objects
$typeData = @{
    TypeName = 'System.IO.FileInfo'
    MemberType = 'AliasProperty'
    MemberName = 'Size'
    Value = 'Length'
}
Update-TypeData @typeData

# Now all files have .Size property
Get-ChildItem | Select-Object Name, Size  # Size is alias for Length

# Add ScriptProperty for dynamic calculation
Update-TypeData -TypeName 'System.IO.FileInfo' -MemberType ScriptProperty -MemberName 'Age' -Value {
    (Get-Date) - $this.LastWriteTime
}

# Add ScriptMethod
Update-TypeData -TypeName 'System.IO.FileInfo' -MemberType ScriptMethod -MemberName 'Compress' -Value {
    $destination = "$( $this.FullName ).zip"
    Compress-Archive -Path $this.FullName -DestinationPath $destination
    return $destination
}
```

**Default Display Property Sets** Control which properties show by default

```powershell
# Define default visible properties for custom type
$typeData = @{
    TypeName = 'My.ServerInfo'
    DefaultDisplayPropertySet = 'Name', 'Status', 'Uptime', 'MemoryUsed'
}
Update-TypeData @typeData

# Create objects with this type name
$server = [PSCustomObject]@{
    PSTypeName = 'My.ServerInfo'  # Critical: sets the type name
    Name = 'Server01'
    Status = 'Online'
    Uptime = '15 days'
    MemoryUsed = '8GB'
    IP = '192.168.1.1'  # Hidden by default
    MAC = '00:11:22:33:44:55'  # Hidden by default
}

# Shows only Name, Status, Uptime, MemoryUsed by default
$server

# Show all properties
$server | Format-List *
```

**Property Sets (Advanced)** Define named property groups for quick selection.

```powershell
# Create ps1xml file for property sets
$ps1xml = @'
<?xml version="1.0" encoding="utf-8"?>
<Types>
  <Type>
    <Name>System.IO.FileInfo</Name>
    <Members>
      <PropertySet>
        <Name>TimeInfo</Name>
        <ReferencedProperties>
          <Name>Name</Name>
          <Name>CreationTime</Name>
          <Name>LastWriteTime</Name>
          <Name>LastAccessTime</Name>
        </ReferencedProperties>
      </PropertySet>
    </Members>
  </Type>
</Types>
'@

$ps1xml | Out-File "$env:TEMP\FileInfo.types.ps1xml"
Update-TypeData -AppendPath "$env:TEMP\FileInfo.types.ps1xml"

# Use the property set
Get-ChildItem | Select-Object TimeInfo
```

## Calculating Custom Properties: Advanced Techniques

**In-Place Calculations with ForEach-Object**

```powershell
# Transform objects with calculated properties
Get-Process | ForEach-Object {
    [PSCustomObject]@{
        ProcessName = $_.Name
        MemoryMB = [math]::Round($_.WorkingSet64 / 1MB, 2)
        CPUPercent = if ($_.CPU)
        {
            [math]::Round($_.CPU / 100, 2)
        }
        else
        {
            0
        }
        IsSystem = $_.SessionId -eq 0
        StartTime = $_.StartTime?.ToString('yyyy-MM-dd HH:mm') ?? 'N/A'
    }
} | Sort-Object MemoryMB -Descending
```

**Grouping with Calculated Properties**

```powershell
# Group by calculated property
Get-Process | Group-Object -Property {
    if ($_.WorkingSet64 -gt 1GB)
    {
        'High Memory'
    }
    elseif ($_.WorkingSet64 -gt 100MB)
    {
        'Medium Memory'
    }
    else
    {
        'Low Memory'
    }
} | Select-Object Name, Count, @{
    Name = 'TotalMemoryMB'
    Expression = {
        [math]::Round(($_.Group | Measure-Object WorkingSet64 -Sum).Sum / 1MB, 2)
    }
}
```

**DateTime Calculations**

```powershell
Get-ChildItem | Select-Object Name, @{
    Name = 'AgeCategory'
    Expression = {
        $days = ((Get-Date) - $_.LastWriteTime).Days
        switch ($days)
        {
            { $_ -le 1 } {
                'Today'
            }
            { $_ -le 7 } {
                'This Week'
            }
            { $_ -le 30 } {
                'This Month'
            }
            default {
                'Older'
            }
        }
    }
}, @{
    Name = 'LastModified'
    Expression = { $_.LastWriteTime.ToString('MMM dd, yyyy HH:mm') }
}
```

### PSTypeName: Type Decoration for Custom Objects

- Mark objects with custom types for validation and formatting

```powershell
# Create typed object
$server = [PSCustomObject]@{
    PSTypeName = 'My.Infrastructure.Server'
    Name = 'Web01'
    IP = '192.168.1.10'
    Role = 'WebServer'
}

# Function validates input type
function Get-ServerStatus
{
    [CmdletBinding()]
    param(
        [Parameter(Mandatory)]
        [PSTypeName('My.Infrastructure.Server')]
        [PSCustomObject]$Server
    )

    Test-Connection -TargetName $Server.IP -Count 1
}

# This works
$server | Get-ServerStatus

# This fails validation
[PSCustomObject]@{ Name = 'Invalid' } | Get-ServerStatus
```

### PSObject.Properties: Dynamic Property Enumeration

- Access and manipulate properties dynamically

```powershell
$object = [PSCustomObject]@{
    Name = 'Server01'
    Status = 'Running'
    Uptime = 15.5
}

# Enumerate all properties
$object.PSObject.Properties | ForEach-Object {
    "$( $_.Name ) = $( $_.Value ) [Type: $( $_.TypeNameOfValue )]"
}

# Check if property exists
$hasProperty = $object.PSObject.Properties.Name -contains 'Status'

# Add property dynamically without Add-Member
$object.PSObject.Properties.Add(
        [System.Management.Automation.PSNoteProperty]::new('NewProperty', 'Value')
)
```

### Ordered Properties

- Maintain property creation order

```powershell
# Standard hashtable (order not guaranteed in PS5.1)
$unordered = @{
    Z = 'Last'
    A = 'First'
    M = 'Middle'
}

# Ordered dictionary (maintains insertion order)
$ordered = [ordered]@{
    Z = 'Last'
    A = 'First'
    M = 'Middle'
}

# PSCustomObject preserves order automatically (PS3+)
$object = [PSCustomObject]$ordered

# Or cast directly (cleanest)
$object = [PSCustomObject]@{
    FirstName = 'John'
    LastName = 'Doe'
    Department = 'IT'
    ID = 12345
}
```

### MemberSet for Default Views

- Control default display behavior

```powershell
# Create object with standard members
$employee = [PSCustomObject]@{
    FirstName = 'Jane'
    LastName = 'Smith'
    Department = 'Engineering'
    Salary = 95000
    SSN = '123-45-6789'
    Password = 'Secret123!'
}

# Define default display set (hides sensitive info)
$defaultDisplay = 'FirstName', 'LastName', 'Department'
$defaultDisplaySet = New-Object System.Management.Automation.PSPropertySet(
'DefaultDisplayPropertySet',
[string[]]$defaultDisplay
)
$PSStandardMembers = [System.Management.Automation.PSMemberInfo[]]@($defaultDisplaySet)

$employee | Add-Member MemberSet PSStandardMembers $PSStandardMembers

# Shows only safe properties by default
$employee

# Show all (including sensitive)
$employee | Format-List *
```

### Formatting Views (Format.ps1xml Alternative)

**Programmatic formatting without XML files**

```powershell
# Define custom table view
Update-FormatData -AppendPath {
    ViewDefinitions = @{
        Name = 'MyServerTable'
        ViewType = 'Table'
        TypeName = 'My.Server'
        Columns = @(
            @{ Label = 'Server'; Width = 15; Expression = { $_.Name } }
            @{ Label = 'Status'; Width = 10; Expression = { $_.Status } }
            @{ Label = 'Memory(GB)'; Width = 12; Expression = { $_.MemoryGB } }
            @{ Label = 'Uptime'; Expression = { $_.Uptime } }
        )
    }
}
```

| Method                | Speed       | Scope               | Persistence      | Best For                   |
|-----------------------|-------------|---------------------|------------------|----------------------------|
| `[PSCustomObject]@{}` | ⭐⭐⭐ Fastest | Single object       | Session          | Creating new objects       |
| `Select-Object`       | ⭐⭐⭐ Fast    | Pipeline            | Session          | Shaping existing objects   |
| `Add-Member`          | ⭐ Slow      | Single object       | Session          | Modifying existing objects |
| `Update-TypeData`     | ⭐⭐ Moderate | All objects of type | Can persist      | Type-wide extensions       |
| `ps1xml files`        | ⭐⭐ Moderate | All objects         | Permanent (file) | Module formatting          |


### Real World Example


```powershell
<#
.SYNOPSIS
    Advanced custom property demonstration with type extensions.
#>
function Get-InventoryReport
{
    [CmdletBinding()]
    [OutputType('My.Inventory.Item')]
    param(
        [string[]]$ComputerName = $env:COMPUTERNAME,
        [switch]$IncludeSensitive
    )

    begin {
        # Define type extensions (once per session)
        if (-not (Get-TypeData -TypeName 'My.Inventory.Item' -ErrorAction SilentlyContinue))
        {
            Update-TypeData -TypeName 'My.Inventory.Item' -MemberType ScriptProperty -MemberName 'AgeDays' -Value {
                ((Get-Date) - $this.LastBootTime).Days
            }

            Update-TypeData -TypeName 'My.Inventory.Item' -DefaultDisplayPropertySet 'ComputerName', 'OS', 'AgeDays', 'Status'
        }

        $results = [System.Collections.Generic.List[object]]::new()
    }

    process {
        foreach ($computer in $ComputerName)
        {
            try
            {
                $cim = New-CimSession -ComputerName $computer -OperationTimeoutSec 30

                $os = Get-CimInstance -CimSession $cim -ClassName Win32_OperatingSystem
                $cs = Get-CimInstance -CimSession $cim -ClassName Win32_ComputerSystem
                $proc = Get-CimInstance -CimSession $cim -ClassName Win32_Processor | Select-Object -First 1

                # Create typed custom object
                $item = [PSCustomObject]@{
                    PSTypeName = 'My.Inventory.Item'
                    ComputerName = $computer
                    OS = $os.Caption
                    OSVersion = $os.Version
                    LastBootTime = $os.LastBootUpTime
                    TotalMemoryGB = [math]::Round($os.TotalVisibleMemorySize / 1MB, 2)
                    Processor = $proc.Name
                    Manufacturer = $cs.Manufacturer
                    Model = $cs.Model
                    SerialNumber = if ($IncludeSensitive)
                    {
                        $cs.Name
                    }
                    else
                    {
                        'REDACTED'
                    }
                    Status = 'Online'
                }

                # Add instance-specific calculated property
                $item | Add-Member -MemberType ScriptProperty -Name 'MemoryStatus' -Value {
                    if ($this.TotalMemoryGB -gt 16)
                    {
                        'Excellent'
                    }
                    elseif ($this.TotalMemoryGB -gt 8)
                    {
                        'Good'
                    }
                    else
                    {
                        'Adequate'
                    }
                } -PassThru

                $results.Add($item)
                Remove-CimSession $cim
            }
            catch
            {
                $results.Add([PSCustomObject]@{
                    PSTypeName = 'My.Inventory.Item'
                    ComputerName = $computer
                    Status = 'Offline'
                    Error = $_.Exception.Message
                })
            }
        }
    }

    end {
        # Return with calculated grouping
        $results | Sort-Object AgeDays -Descending
    }
}

# Usage
$inventory = Get-InventoryReport -ComputerName 'Server01', 'Server02', 'Server03'
$inventory | Format-Table  # Uses default display set
$inventory | Format-List *  # Shows all properties
$inventory | Where-Object { $_.AgeDays -gt 30 }  # Use typed property
```
