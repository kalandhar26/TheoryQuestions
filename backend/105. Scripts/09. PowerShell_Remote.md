# Setting Up PowerShell Remoting

**Enabling Remoting (WinRM)**

```powershell
# Enable PowerShell remoting on target machine (run as admin)
Enable-PSRemoting -Force

# Verify configuration
Get-PSSessionConfiguration

# Configure trusted hosts (for non-domain environments)
Set-Item WSMan:\localhost\Client\TrustedHosts -Value "Server01,Server02,192.168.1.*" -Concatenate -Force

# View current trusted hosts
Get-Item WSMan:\localhost\Client\TrustedHosts
```

**Security Best Practices**

        Use HTTPS: Configure SSL certificates for WinRM
        Limit Access: Restrict to specific users/groups via Group Policy
        Enable JEA: Use Just Enough Administration for privileged access
        Firewall: WinRM uses ports 5985 (HTTP) and 5986 (HTTPS)

## Invoke-Command vs PS Sessions

**Invoke-Command (One-off Execution)**
Best for single commands or parallel execution across multiple systems

```powershell
# Single command to one computer
Invoke-Command -ComputerName "Server01" -ScriptBlock { Get-Process }

# Multiple computers in parallel
Invoke-Command -ComputerName "Server01", "Server02", "Server03" -ScriptBlock {
    Get-CimInstance Win32_OperatingSystem | Select-Object Caption, LastBootUpTime
}

# With parameters
$serviceName = "w32time"
Invoke-Command -ComputerName "Server01" -ScriptBlock {
    param($name)
    Get-Service -Name $name
} -ArgumentList $serviceName

# Using $using: for local variables (PowerShell 3+)
$localVar = "Important Data"
Invoke-Command -ComputerName "Server01" -ScriptBlock {
    "Received: $using:localVar"
}

# Return objects automatically serialize
$results = Invoke-Command -ComputerName "Server01", "Server02" -ScriptBlock {
    [PSCustomObject]@{
        ComputerName = $env:COMPUTERNAME
        Time = Get-Date
        Services = (Get-Service | Where-Object { $_.Status -eq 'Running' }).Count
    }
}
$results | Format-Table PSComputerName, Time, Services
```

**Persistent Sessions (Multiple Commands)**

- Use New-PSSession when you need to run multiple commands or maintain state

```powershell
# Create persistent session
$session = New-PSSession -ComputerName "Server01" -Credential (Get-Credential)

# Execute multiple commands in same session
Invoke-Command -Session $session -ScriptBlock { $var = "Persistent Data" }
Invoke-Command -Session $session -ScriptBlock { "Variable still exists: $var" }

# Interactive session (like SSH)
Enter-PSSession -Session $session
[Server01]: PS C: \> Get-Process  # Now on remote machine
[Server01]: PS C: \> Exit-PSSession  # Return to local

# Multiple sessions for parallel operations
$sessions = New-PSSession -ComputerName "Server01", "Server02", "Server03"
Invoke-Command -Session $sessions -ScriptBlock {
Get-Process
}

# Cleanup
Remove-PSSession -Session $sessions
Get-PSSession | Remove-PSSession  # Remove all
```

| Approach                           | Best For                             | Performance                   |
|------------------------------------|--------------------------------------|-------------------------------|
| `Invoke-Command` (direct)          | Single command, one-time execution   | Fastest (no session overhead) |
| `New-PSSession` + `Invoke-Command` | Multiple commands, state persistence | Good for repeated operations  |
| `Enter-PSSession`                  | Interactive troubleshooting          | N/A (interactive only)        |
| `Invoke-Command -AsJob`            | Long-running background tasks        | Non-blocking                  |

## Just Enough Administration (JEA)

- JEA provides constrained, audited endpoints for delegated administration

```powershell
# 1. Create role capability file (defines what users can do)
$roleCapability = @{
    VisibleCmdlets = @(
        'Get-Service',
        'Restart-Service',
        'Get-Process',
        'Get-EventLog'
    )
    VisibleFunctions = @()
    VisibleExternalCommands = @()
}
New-PSRoleCapabilityFile -Path "C:\Program Files\WindowsPowerShell\Modules\MyJEA\RoleCapabilities\HelpDesk.psrc" -VisibleCmdlets $roleCapability.VisibleCmdlets

# 2. Create session configuration file
$sessionConfig = @{
    SessionType = 'RestrictedRemoteServer'
    RunAsVirtualAccount = $true
    TranscriptDirectory = 'C:\ProgramData\JEATranscripts'
    RoleDefinitions = @{
        'CONTOSO\HelpDesk' = @{
            RoleCapabilities = 'HelpDesk'
        }
    }
}
New-PSSessionConfigurationFile -Path "C:\ProgramData\HelpDeskJEA.pssc" @sessionConfig

# 3. Register the JEA endpoint
Register-PSSessionConfiguration -Name 'HelpDeskJEA' -Path 'C:\ProgramData\HelpDeskJEA.pssc' -Force

# 4. Connect as limited user
Enter-PSSession -ComputerName Server01 -ConfigurationName HelpDeskJEA -Credential (Get-Credential)
# User can only run Get-Service, Restart-Service, Get-Process, Get-EventLog
```

## Creating Custom Objects with PSCustomObject

```powershell
# Basic creation - preserves order and fastest performance
$server = [PSCustomObject]@{
    Name = 'Server01'
    IP = '192.168.1.10'
    Role = 'WebServer'
    Status = 'Online'
    LastCheck = Get-Date
}

# With calculated properties
$process = Get-Process -Name powershell | Select-Object -First 1
$customProcess = [PSCustomObject]@{
    ProcessName = $process.ProcessName
    ID = $process.Id
    MemoryMB = [math]::Round($process.WorkingSet64 / 1MB, 2)
    StartTime = $process.StartTime?.ToString('yyyy-MM-dd HH:mm') ?? 'N/A'
    IsElevated = ([Security.Principal.WindowsPrincipal][Security.Principal.WindowsIdentity]::GetCurrent()).IsInRole([Security.Principal.WindowsBuiltInRole]::Administrator)
}
```

```powershell
| Method                              | Relative Speed                       | Use Case                        |
| ----------------------------------- | ------------------------------------ | ------------------------------- |
| `[PSCustomObject]@{
}`               | ⭐⭐⭐ **Fastest** (0.94s/5000 objects) | New objects, all scenarios      |
| `New-Object PSObject -Property @{
}` | ⭐⭐ Moderate (7.29s/5000 objects)     | Legacy compatibility            |
| `Select-Object`                     | ⭐⭐ Moderate (7.74s/5000 objects)     | Shaping existing objects        |
| **Add-Member**                      | ⭐ **Slowest** (28.87s/5000 objects)  | Modifying existing objects only |

```

## Adding Script Properties and Methods

**Using Add-Member (Instance-Level)**

- Adds functionality to single object

```powershell
$server = [PSCustomObject]@{
    Name = 'Server01'
    IP = '192.168.1.10'
}

# Add NoteProperty (static value)
$server | Add-Member -MemberType NoteProperty -Name 'Location' -Value 'Datacenter A' -PassThru

# Add ScriptProperty (dynamic calculation)
$server | Add-Member -MemberType ScriptProperty -Name 'IsOnline' -Value {
    Test-Connection -TargetName $this.IP -Count 1 -Quiet
} -PassThru

# Add ScriptMethod (action)
$server | Add-Member -MemberType ScriptMethod -Name 'Restart' -Value {
    Write-Host "Restarting $( $this.Name )..."
    # Restart-Computer -ComputerName $this.Name -Force
} -PassThru

# Add AliasProperty (shortcut)
$server | Add-Member -MemberType AliasProperty -Name 'Hostname' -Value 'Name' -PassThru

# Usage
$server.IsOnline  # Dynamic check
$server.Restart() # Execute method
$server.Hostname  # Same as $server.Name
```

### Using Update-TypeData (Type-Level)

- Adds functionality to all objects of a type

```powershell
# Define type name in object
$server = [PSCustomObject]@{
    PSTypeName = 'My.Server'  # Critical for type identification
    Name = 'Server01'
    IP = '192.168.1.10'
}

# Add script property to ALL objects of this type
Update-TypeData -TypeName 'My.Server' -MemberType ScriptProperty -MemberName 'Uptime' -Value {
    if ($this.IsOnline)
    {
        $os = Get-CimInstance -ComputerName $this.Name -ClassName Win32_OperatingSystem
        (Get-Date) - $os.LastBootUpTime
    }
    else
    {
        $null
    }
}

# Add script method to ALL objects of this type
Update-TypeData -TypeName 'My.Server' -MemberType ScriptMethod -MemberName 'GetServices' -Value {
    param([string]$Status = 'Running')
    Get-Service -ComputerName $this.Name | Where-Object { $_.Status -eq $Status }
}

# Now all My.Server objects have these members
$server2 = [PSCustomObject]@{
    PSTypeName = 'My.Server'
    Name = 'Server02'
    IP = '192.168.1.11'
}
$server2.Uptime  # Works automatically
$server2.GetServices('Stopped')  # Works automatically
```

**Default Display Property Sets**

- Control which properties show by default

```powershell
# Create object with many properties
$server = [PSCustomObject]@{
    PSTypeName = 'My.Server'
    Name = 'Server01'
    IP = '192.168.1.10'
    OS = 'Windows Server 2022'
    SerialNumber = 'ABC123456'
    AdminPassword = 'Secret!'  # Sensitive
    LastCheck = Get-Date
}

# Define default visible properties
Update-TypeData -TypeName 'My.Server' -DefaultDisplayPropertySet 'Name', 'IP', 'OS', 'LastCheck'

# Shows only safe properties by default
$server

# Show all properties including sensitive
$server | Format-List *
```

### $using: Scope Modifier

- Pass local variables to remote sessions

```powershell
$localPath = "C:\Local\File.txt"
$remotePath = "C:\Remote\File.txt"

Invoke-Command -ComputerName "Server01" -ScriptBlock {
    # Without $using: - $localPath doesn't exist here
    # With $using: - accesses parent's $localPath
    Copy-Item -Path $using:localPath -Destination $using:remotePath
}
```

### ArgumentList vs $using:

```powershell
# Multiple parameters - cleaner with ArgumentList
$params = @{
    Path = "C:\Logs"
    Days = 30
    Pattern = "*.log"
}

Invoke-Command -ComputerName "Server01" -ScriptBlock {
    param($p)
    Get-ChildItem -Path $p.Path -Filter $p.Pattern |
            Where-Object { $_.LastWriteTime -lt (Get-Date).AddDays(-$p.Days) }
} -ArgumentList $params

# Or use $using: with hashtable
Invoke-Command -ComputerName "Server01" -ScriptBlock {
    Get-ChildItem -Path $using:params.Path -Filter $using:params.Pattern
}
```

### Disconnect and Reconnect Sessions

```powershell
# Create session and disconnect (keeps running on remote)
$session = New-PSSession -ComputerName "Server01"
Invoke-Command -Session $session -ScriptBlock {
    Start-Process -FilePath "long-running-job.exe"
} -AsJob

Disconnect-PSSession -Session $session  # Session persists on remote

# Later - reconnect from different machine or after network interruption
Get-PSSession -ComputerName "Server01" | Connect-PSSession
Enter-PSSession -Id $session.Id
```

### Copy-Item with Sessions (PSDirect)

```powershell
$session = New-PSSession -ComputerName "Server01"

# Copy to remote
Copy-Item -Path "C:\Local\script.ps1" -Destination "C:\Remote\" -ToSession $session

# Copy from remote
Copy-Item -Path "C:\Remote\results.csv" -Destination "C:\Local\" -FromSession $session

# Copy entire directory
Copy-Item -Path "C:\Local\Tools\" -Destination "C:\Remote\" -Recurse -ToSession $session
```

### Invoke-Command with Local Functions

```powershell
# Define function locally
function Get-SystemHealth
{
    Get-CimInstance Win32_OperatingSystem | Select-Object @{ N = 'MemoryUsage'; E = { [math]::Round(($_.TotalVisibleMemorySize - $_.FreePhysicalMemory) / $_.TotalVisibleMemorySize * 100, 2) } }
}

# Export and run remotely
$servers = "Server01", "Server02", "Server03"
Invoke-Command -ComputerName $servers -ScriptBlock ${function:Get-SystemHealth}
```

### PSObject.Properties Manipulation

- Direct property manipulation without Add-Member

```powershell
$object = [PSCustomObject]@{
    Name = 'Server01'
    Status = 'Running'
}

# Enumerate properties
$object.PSObject.Properties | ForEach-Object {
    "$( $_.Name ) = $( $_.Value )"
}

# Check if property exists
$hasProperty = $object.PSObject.Properties.Name -contains 'Status'

# Remove property
$object.PSObject.Properties.Remove('Status')

# Clone object (shallow copy)
$newObject = $object.PSObject.Copy()
```

| Topic                         | Importance | Description                                      |
|-------------------------------|------------|--------------------------------------------------|
| **`$using:` scope**           | ⭐ Critical | Pass local variables to remote sessions          |
| **Disconnect/Reconnect**      | High       | Persistent sessions across network interruptions |
| **Copy-Item with sessions**   | High       | Efficient file transfer via remoting             |
| **Function export to remote** | Medium     | `${function:Name}` syntax                        |
| **PSObject.Properties**       | Medium     | Direct property enumeration and manipulation     |
| **JEA endpoints**             | ⭐ Critical | Secure, audited, constrained administration      |
| **ArgumentList**              | Medium     | Parameter passing alternative to `$using:`       |

## Real World Example

```powershell
<#
.SYNOPSIS
    Remote server management with custom objects and JEA.
#>
function Get-RemoteServerInventory
{
    [CmdletBinding()]
    param(
        [Parameter(Mandatory)]
        [string[]]$ComputerName,

        [switch]$UseJEA,

        [string]$JEAEndpoint = 'HelpDeskJEA'
    )

    begin {
        # Create reusable sessions for performance
        $sessionParams = @{
            ComputerName = $ComputerName
            ErrorAction = 'SilentlyContinue'
        }
        if ($UseJEA)
        {
            $sessionParams.ConfigurationName = $JEAEndpoint
        }
        $sessions = New-PSSession @sessionParams

        # Define custom object type with methods
        Update-TypeData -TypeName 'RemoteServer.Inventory' -MemberType ScriptMethod -MemberName 'TestConnection' -Value {
            Test-Connection -TargetName $this.ComputerName -Count 1 -Quiet
        } -Force

        Update-TypeData -TypeName 'RemoteServer.Inventory' -DefaultDisplayPropertySet 'ComputerName', 'Status', 'OS', 'LastBoot' -Force
    }

    process {
        $results = Invoke-Command -Session $sessions -ScriptBlock {
            $os = Get-CimInstance Win32_OperatingSystem -ErrorAction SilentlyContinue
            $cs = Get-CimInstance Win32_ComputerSystem -ErrorAction SilentlyContinue

            if ($os)
            {
                [PSCustomObject]@{
                    PSTypeName = 'RemoteServer.Inventory'
                    ComputerName = $env:COMPUTERNAME
                    Status = 'Online'
                    OS = $os.Caption
                    LastBoot = $os.LastBootUpTime
                    UptimeDays = [math]::Floor(((Get-Date) - $os.LastBootUpTime).TotalDays)
                    TotalMemoryGB = [math]::Round($cs.TotalPhysicalMemory / 1GB, 2)
                    Manufacturer = $cs.Manufacturer
                    Model = $cs.Model
                    SerialNumber = $cs.Name  # Simplified for demo
                }
            }
            else
            {
                [PSCustomObject]@{
                    PSTypeName = 'RemoteServer.Inventory'
                    ComputerName = $env:COMPUTERNAME
                    Status = 'Offline'
                    Error = 'Failed to query CIM'
                }
            }
        }

        # Add instance-specific script properties
        foreach ($item in $results)
        {
            if ($item.Status -eq 'Online')
            {
                $item | Add-Member -MemberType ScriptProperty -Name 'MemoryStatus' -Value {
                    if ($this.TotalMemoryGB -gt 16)
                    {
                        'Good'
                    }
                    elseif ($this.TotalMemoryGB -gt 8)
                    {
                        'Adequate'
                    }
                    else
                    {
                        'Low'
                    }
                } -PassThru -Force
            }
        }

        return $results
    }

    end {
        # Cleanup sessions
        $sessions | Remove-PSSession
    }
}

# Usage
$inventory = Get-RemoteServerInventory -ComputerName 'Server01', 'Server02', 'Server03'
$inventory | Format-Table  # Uses default display set
$inventory | Where-Object { $_.TestConnection() } | Format-List *  # Uses type method
```
