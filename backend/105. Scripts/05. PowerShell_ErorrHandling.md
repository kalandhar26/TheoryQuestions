## Error Action Preference Variables

### $ErrorActionPreference (Global Setting)

- Controls how PowerShell responds to non-terminating errors across your entire session or script.
- $ErrorActionPreference applies to both non-terminating AND statement-terminating errors, while -ErrorAction parameter
  only affects non-terminating errors.

```powershell
# Valid values: Stop, Continue (default), SilentlyContinue, Inquire, Break
$ErrorActionPreference = "Stop"  # Convert all non-terminating errors to terminating

# Best practice: Save and restore original value
$originalPreference = $ErrorActionPreference
try
{
    $ErrorActionPreference = "Stop"
    # Critical code here
}
finally
{
    $ErrorActionPreference = $originalPreference
}
```

| Variable                                   | Purpose                                   | Default                                  |
|--------------------------------------------|-------------------------------------------|------------------------------------------|
| `$ErrorView`                               | Error display format                      | `ConciseView` (PS7) / `NormalView` (PS5) |
| `$WarningPreference`                       | Warning action                            | `Continue`                               |
| `$VerbosePreference`                       | Verbose output                            | `SilentlyContinue`                       |
| `$DebugPreference`                         | Debug messages                            | `SilentlyContinue`                       |
| `$InformationPreference`                   | Information stream                        | `SilentlyContinue`                       |
| `$PSNativeCommandUseErrorActionPreference` | Treat native command exit codes as errors | `$false`                                 |

```powershell
# PowerShell 7+ concise error view (new default)
$ErrorView = 'ConciseView'  # Short, readable errors
$ErrorView = 'NormalView'   # Traditional detailed errors

# PowerShell 7.2+ custom views
$ErrorView = 'DetailedView' # More technical details
$ErrorView = 'CategoryView' # Category-based summary
```

### The -ErrorAction Parameter

- Overrides `$ErrorActionPreference` for a specific command.

```powershell
# Common parameter available on all cmdlets
Get-Content "missing.txt" -ErrorAction SilentlyContinue  # Suppress error
Get-Content "missing.txt" -ErrorAction Stop              # Make terminating (catchable)
Get-Content "missing.txt" -ErrorAction Inquire           # Prompt user
Get-Content "missing.txt" -ErrorAction Ignore            # Ignore completely (not logged!)

# Aliases
Get-Content "missing.txt" -EA SilentlyContinue
Get-Content "missing.txt" -EA 0  # 0=SilentlyContinue, 1=Stop, 2=Continue, 3=Inquire, 4=Ignore
```

- `$ErrorActionPreference` Global setting for the session/script
- `-ErrorAction` Per-command override (doesn't affect statement-terminating errors)

## Try, Catch, Finally Blocks

```powershell
try
{
    # Code that might throw terminating error
    Get-Content "critical.txt" -ErrorAction Stop
    Invoke-RestMethod -Uri "https://api.example.com/data" -ErrorAction Stop
}
catch
{
    # Catches ALL errors (generic handler)
    Write-Host "Error occurred: $_" -ForegroundColor Red
    Write-Host "Exception type: $( $_.Exception.GetType().FullName )"
}
finally
{
    # ALWAYS executes (cleanup, logging, close connections)
    Write-Host "Cleanup completed" -ForegroundColor Cyan
}
```

```powershell
function Write-ErrorLog
```

## API Error Handling

```powershell
function Invoke-ApiCall
{
    param([string]$Uri, [hashtable]$Headers, [int]$MaxRetries = 3)

    $attempt = 0
    do
    {
        $attempt++
        try
        {
            $response = Invoke-RestMethod -Uri $Uri -Headers $Headers -ErrorAction Stop
            return $response
        }
        catch [System.Net.WebException]
        {
            $statusCode = $_.Exception.Response.StatusCode.value__

            if ($statusCode -eq 429 -and $attempt -lt $MaxRetries)
            {
                # Rate limited - retry with backoff
                $delay = [math]::Pow(2, $attempt)
                Write-Warning "Rate limited. Retrying in $delay seconds..."
                Start-Sleep -Seconds $delay
                continue
            }
            elseif ($statusCode -eq 401)
            {
                Write-Error "Authentication failed. Check API key."
                throw  # Don't retry auth failures
            }
            elseif ($statusCode -eq 404)
            {
                Write-Warning "Resource not found at $Uri"
                return $null  # Graceful handling
            }
            else
            {
                Write-Error "HTTP $statusCode : $( $_.Exception.Message )"
                throw
            }
        }
        catch [System.Management.Automation.ValidationMetadataException]
        {
            Write-Error "Invalid URI format: $Uri"
            throw
        }
        catch
        {
            Write-Error "Unexpected API error: $_"
            throw
        }
    } while ($attempt -lt $MaxRetries)

    throw "Max retries exceeded"
}
```

### Catching $LASTEXITCODE

- Native Command Error Handling

```powershell
    1 = 'One or more files were copied successfully.'

    4 = 'Some mismatched files or directories were detected.'
    8 = 'Some files or directories could not be copied.'
```

## Advanced Error Handling

### The trap Statement

- Scope-wide error handler for terminating errors outside of try-catch

```powershell
trap [System.Management.Automation.CommandNotFoundException]
{
    Write-Host "Command not found trapped: $( $_.Exception.Message )" -ForegroundColor Yellow
    continue  # Continue execution after trap
}

trap [System.Exception]
{
    Write-Host "Generic trap caught: $_" -ForegroundColor Red
    break     # Terminate execution (default behavior)
}

# This will trigger the trap
ThisCommandDoesNotExist

Write-Host "This line may or may not execute depending on trap action"
```

**Trap Scopes:**

    Traps are scope-specific (function, script, or global)
    Child scopes inherit parent traps unless overridden
    break in trap = terminate, continue = resume after error

### $PSCmdlet.ThrowTerminatingError()

- Proper way to throw terminating errors in advanced functions

```powershell
function Test-Advanced
{
    [CmdletBinding()]
    param([string]$Path)

    if (-not (Test-Path $Path))
    {
        $errorRecord = [System.Management.Automation.ErrorRecord]::new(
                [System.IO.FileNotFoundException]::new("File not found: $Path"),
                "PathNotFound",
                "ObjectNotFound",

    # $PSCmdlet.ThrowTerminatingError("File not found: $Path")
```

    Respects -ErrorAction and $ErrorActionPreference
    Properly integrates with PowerShell's error system
    Better for reusable modules and functions

### Error Handling in Pipelines

```powershell
# Continue on error for pipeline items
$files = "file1.txt", "file2.txt", "file3.txt"
$results = $files | ForEach-Object {
    try
    {
        Get-Content $_ -ErrorAction Stop
        [PSCustomObject]@{ File = $_; Status = "Success"; Content = $_ }
    }
    catch
    {
        [PSCustomObject]@{ File = $_; Status = "Failed"; Error = $_.Exception.Message }
    }
}

# Filter successful results
$successful = $results | Where-Object { $_.Status -eq "Success" }

# Report failures
$failures = $results | Where-Object { $_.Status -eq "Failed" }

}
```

| Practice                 | Recommendation                                                          |
|--------------------------|-------------------------------------------------------------------------|
| **Global Settings**      | Set `$ErrorActionPreference = "Stop"` at script start for strict mode   |
| **Per-Command Override** | Use `-ErrorAction SilentlyContinue` for expected, non-critical errors   |
| **Try-Catch Usage**      | Always use `-ErrorAction Stop` to make non-terminating errors catchable |
| **Specific Exceptions**  | Catch specific exception types before generic catch blocks              |
| **Logging**              | Use `-ErrorVariable` or custom functions to log without stopping        |
| **Native Commands**      | Check `$LASTEXITCODE` immediately after external commands               |
| **Cleanup**              | Use `finally` blocks for resource cleanup (connections, files, locks)   |
| **Re-throwing**          | Use `throw` (not `Throw $_`) to preserve stack trace                    |
| **Custom Errors**        | Use `$PSCmdlet.ThrowTerminatingError()` in advanced functions           |

### Real World Example

```powershell
<#
.SYNOPSIS
    Production-ready script demonstrating comprehensive error handling.
#>
[CmdletBinding()]
param(
    [Parameter(Mandatory)]
    [string]$ConfigPath,

    [string]$LogPath = "C:\Logs\operations.log",

    [ValidateRange(1, 10)]
    [int]$MaxRetries = 3
)

# Initialize strict error handling
$ErrorActionPreference = "Stop"
$PSNativeCommandUseErrorActionPreference = $true  # PS7+ native command handling

# Custom error class for business logic
class BusinessRuleException: System.Exception
{
    BusinessRuleException([string]$message): base($message)
    {
    }
}

function Write-StructuredLog
{
    param([string]$Level, [string]$Message, [System.Exception]$Exception = $null)

    $logEntry = [PSCustomObject]@{
        Timestamp = Get-Date -Format "yyyy-MM-ddTHH:mm:ss.fff"
        Level = $Level
        Message = $Message
        ScriptName = $MyInvocation.ScriptName
        LineNumber = $MyInvocation.ScriptLineNumber
        ExceptionType = $Exception?.GetType().Name
        ExceptionMessage = $Exception?.Message
    } | ConvertTo-Json -Compress

    Add-Content -Path $LogPath -Value $logEntry -ErrorAction SilentlyContinue
}

try
{
    Write-StructuredLog -Level "INFO" -Message "Starting processing with config: $ConfigPath"

    # Validate configuration
    if (-not (Test-Path $ConfigPath))
    {
        throw [System.IO.FileNotFoundException]::new("Configuration file not found", $ConfigPath)
    }

    $config = Get-Content $ConfigPath -Raw | ConvertFrom-Json

    # Business logic validation
    if ($config.Version -lt 2.0)
    {
        throw [BusinessRuleException]::new("Config version $( $config.Version ) is not supported. Minimum required: 2.0")
    }

    # Process with retry logic
    $attempt = 0
    $success = $false

    do
    {
        $attempt++
        try
        {
            # Simulated API call
            $response = Invoke-RestMethod -Uri $config.ApiEndpoint -Headers @{
                Authorization = "Bearer $( $config.Token )"
            } -TimeoutSec 30 -ErrorAction Stop

            $success = $true
            Write-StructuredLog -Level "INFO" -Message "API call successful on attempt $attempt"
        }
        catch [System.Net.WebException]
        {
            $status = $_.Exception.Response?.StatusCode
            Write-StructuredLog -Level "WARN" -Message "Attempt $attempt failed with HTTP $status" -Exception $_.Exception

            if ($attempt -lt $MaxRetries -and $status -in @(429, 503, 504))
            {
                $delay = [math]::Pow(2, $attempt)
                Write-Host "Retrying in $delay seconds..." -ForegroundColor Yellow
                Start-Sleep -Seconds $delay
            }
            else
            {
                throw  # Re-throw if not retryable or max retries exceeded
            }
        }
    } while (-not $success -and $attempt -lt $MaxRetries)

    if (-not $success)
    {
        throw "Failed to complete operation after $MaxRetries attempts"
    }

    # Native command with exit code handling
    robocopy $config.SourcePath $config.DestPath /MIR /NP /NFL /NDL
    if ($LASTEXITCODE -ge 8)
    {
        throw "Robocopy failed with exit code $LASTEXITCODE"
    }

    Write-StructuredLog -Level "INFO" -Message "Processing completed successfully"
}
catch [System.IO.FileNotFoundException]
{
    Write-StructuredLog -Level "ERROR" -Message "Configuration error" -Exception $_.Exception
    Write-Error "Setup Error: $( $_.Exception.Message )"
    exit 1
}
catch [BusinessRuleException]
{
    Write-StructuredLog -Level "ERROR" -Message "Business rule violation" -Exception $_.Exception
    Write-Error "Validation Error: $( $_.Exception.Message )"
    exit 2
}
catch [System.Net.WebException]
{
    Write-StructuredLog -Level "ERROR" -Message "API communication failed" -Exception $_.Exception
    Write-Error "Network Error: $( $_.Exception.Message )"
    exit 3
}
catch
{
    Write-StructuredLog -Level "ERROR" -Message "Unexpected error" -Exception $_.Exception
    Write-Error "Critical Error: $_"
    exit 99
}
finally
{
    # Cleanup regardless of success/failure
    if (Test-Path variable:\config)
    {
        Remove-Variable config -ErrorAction SilentlyContinue
    }
    Write-StructuredLog -Level "INFO" -Message "Cleanup completed, script ending with exit code $LASTEXITCODE"
}
```

## Terminating vs Non-Terminating Errors

- A terminating error stops execution immediately.
- A Non-terminating error keeps powershell to continue and its default behavior for most cmdlets.
- `$ErrorActionPreference` is variable used to have an effect on every script. It allows us to set error handling
  behavior for individual commands, giving us more fine-grained control over scripts.
- `$ErrorActionPreference` set a default behavior for our entire script or session.
- `-ErrorAction` parameter set the behavior of individual cmdlets.
- `continue` (Display error message and continue), `SilentlyContinue` (Suppress error message and continue), `ignore` (
  Ignore the errors), `stop` ( Stop the execution of script at first issue).
- We need to balance catching critical errors that should halt our script and ignore minor issues that should not
  prevent our script from completing task. The choice of `$ErrorActionPreference` depends on that script for that
  moment.
- We can set global preference with `$ErrorActionPreference` and then override it for specific commands using
  `-ErrorAction`
- The script should stop if it encounters an error, or should it keep trying to monitor the folder.
- Check the cpu monitoring script in do-while (file_organizer.ps1)

```powershell
{
    # Construct the path for destination foler
    # $PWD represents the current working directory, and we are creating a subfolder for each category within it
    $destinationFolder = "$PWD\$category"

    # Create the destination folder if it does not exist and move the file to the appropriate category folder
    # The [void] supress the output of New-Itenm cmdlet, and Out-Null is used to discard the output of Move-Item cmdlet

    {

    # This represents the original file name.

    # Remove the -WhatIf parameter to actually move the files instead of simulating the move operation
```
